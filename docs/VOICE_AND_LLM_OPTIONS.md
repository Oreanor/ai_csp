# Речь (STT/TTS) и текстовые LLM: варианты с упором на португальский и цену

Краткий ориентир для PoC: что дешевле, что попробовать бесплатно, почему с **португальским распознаванием речи** часто боль, и как это разделить с **генерацией ответов** и **озвучкой**.

## Почему PT в речи сложнее

- **Два больших стандарта**: бразильский (pt-BR) и европейский (pt-PT) — разная фонетика, лексика, интонация. Модель «для Portuguese» без уточнения региона даёт смешанное качество.
- **Акцент и шум**: телефонный/ноутбучный микрофон, эхо, фоновая речь — WER растёт сильнее, чем для английского в тех же условиях.
- **Разделение задач**: качество **распознавания** (STT) и качество **диалога** (LLM) — разные сервисы; улучшение одного не заменяет другое.

Практика: в продукте явно выбирать **pt-BR** или **pt-PT**, прогонять **ваши** записи (2–5 мин интервью) по shortlist STT и смотреть WER/субъективно, а не ориентироваться только на «поддерживает PT».

## Речь → текст (STT), особенно португальский

### Бесплатно / почти без денег (для экспериментов)

| Подход | Плюсы | Минусы |
|--------|--------|--------|
| **Whisper large-v3 локально** (`faster-whisper`, `whisper.cpp`, интеграция через Python/CLI) | Нет платы за минуту; сильный **мультиязычный** baseline, хорошо знаком с PT в массе данных | Нужен GPU/CPU и пайплайн; **не стриминг** в классическом виде; латентность на длинных файлах |
| **Whisper через облако** (OpenAI API, Azure batch и т.д.) | Меньше возни с железом | Платно по минутам после бесплатных кредитов; лимиты файлов |
| **Браузерный Web Speech API** | Ноль API-ключей | Качество и языки **зависят от браузера/OS**; для PT часто слабее облака |

Имеет смысл начать с **локального Whisper** на реальных фразах интервьюера и кандидата (BR vs PT явно в настройках языка).

### Относительно дешёвые облачные STT (часто лучше PT, чем «голый» браузер)

- **Google Cloud Speech-to-Text** (в т.ч. модели вроде Chirp / long) — широкое покрытие языков, для продакшена обычно в shortlist.
- **Microsoft Azure AI Speech** — streaming, кастомный словарь, хорошая интеграция с остальным стеком Microsoft; для **реального времени** часто предпочтительнее batch-Whisper.
- **Deepgram, AssemblyAI** — удобные streaming API, конкурентные цены; точность по PT проверять на своих данных.
- **OpenAI Whisper API** — предсказуемый API, плата за минуту.

Публичные бенчмарки (Soniox и др., 2025) показывают, что **рейтинг по языкам сильно прыгает**: для **Portuguese** в одном отчёте хорошо выглядят **Soniox / AssemblyAI / Speechmatics / Whisper**, у **Google** в той же таблице для PT был высокий WER — это аргумент **не верить одной цифре**, а тестировать **ваш** сценарий.

Итог: для **PT и денег** разумный путь PoC — **локальный Whisper** + один **облачный** кандидат (Google или Azure) для сравнения на одних и тех же аудио.

## Текст → ответ (LLM) для интервью

Здесь язык UI — **текст**; PT «ломается» реже, чем в STT, если модель нормально знает pt-BR/pt-PT в промпте.

### Бесплатно / дёшево для старта

| Вариант | Комментарий |
|---------|-------------|
| **Google AI Studio (Gemini API)** | Щедрый **бесплатный** tier для прототипов (лимиты RPM/TPM и политика использования данных — читать актуальные условия). Удобно для **Flash**-класса моделей. |
| **Groq** | **Без карты** на free tier; очень быстрый инференс для **Llama** / части открытых моделей; лимиты TPM/RPM — хватит на dev. |
| **Ollama (локально)** | **0 ₽** за токен, плата только электричество/железо; OpenAI-совместимый endpoint на `localhost`. Подходит для офлайн и тестов без облака. |
| **OpenAI** | **Нет постоянного free tier**; для экономии — **gpt-4o-mini** и жёсткие лимиты `max_tokens`. |

Для «дешевле в облаке» после PoC: **Gemini Flash**, **GPT-4o-mini**, малые модели на **Groq** — обычно дешевле «флагманов» при приемлемом качестве для коротких реплик интервью.

## Текст → речь (TTS) для кандидата

| Уровень | Варианты |
|---------|----------|
| **Бесплатно / офлайн** | **Piper** (лёгкий нейро TTS, есть голоса под разные языки); системный **speechSynthesis** в браузере — бесплатно, но качество «робот». |
| **Дешёвое облако** | **Azure Neural**, **Google Cloud Text-to-Speech**, **Amazon Polly** — низкая цена за миллион символов относительно «премиум» студийных голосов. |
| **Дороже, но выразительно** | **ElevenLabs** и аналоги — часто лучше эмоции/естественность; есть ограниченный free tier на старте, далее плата. |

Для PT снова важно: **голос и языковая модель TTS** должны совпадать с **pt-BR** или **pt-PT**.

---

## Как собрать пайплайн в продукте

1. **STT**: выбрать 1 локальный (Whisper) + 1 облачный → замерить на PT-BR записях.  
2. **LLM**: `LlmClient` в репозитории — подключить сначала **Gemini** или **Groq** (дешево/бесплатно для dev).  
3. **TTS**: отдельный клиент (не путать с LLM); начать с **Piper** или **облачного нейро** с явным `pt-BR`.

Цены и лимиты free tier меняются — перед выбором сверяйте официальные прайсы и ToS.
